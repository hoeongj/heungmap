# 06. 참가신청서 작성 초안 (그대로 제출용)

> 접수: 한국관광 콘텐츠랩 https://api.visitkorea.or.kr (회원가입 필수) · **마감 2026-07-21 16:00**

## 필수 입력 항목

**서비스 유형**: ☑ **웹** (단일)
- 앱까지 체크하면 "동일 내용 2종 개발" 의무 발생 → 초심자 팀엔 부담. 웹 하나로.

**서비스명**: **흥할지도 (HeungMap)**

**서비스 개요**:
> 정확한 수요 예측으로 기획자와 방문객 모두를 만족시키는 데이터 기반 축제 플랫폼입니다. 한국관광공사 TourAPI(축제·관광 정보)와 지역별 방문자 빅데이터를 학습한 예측 모델로, 기획자에게는 예상 방문객 수를 예측해 개최지·주차·숙박 등 편의시설 최적 구성과 기획안·보완 피드백을 제공하고, 방문객에게는 예상 혼잡도·티켓팅 경쟁도와 주변 주차·숙소 정보 등 축제를 더 즐길 수 있는 방법을 제안합니다.

> 📌 접수 폼이 "1~2문장"만 받으면 위 문장을 그대로 써도 2문장이라 OK. 더 줄여야 하면:
> "한국관광공사 TourAPI와 지역별 방문자 빅데이터를 학습한 예측 모델로 축제 수요를 예측해, 기획자에게는 개최지·주차·숙박 등 편의시설 최적 구성과 보완 피드백을, 방문객에게는 예상 혼잡도·티켓팅 경쟁도와 주변 주차·숙소 정보를 제공하는 데이터 기반 축제 플랫폼입니다."

## 이어서 나올 법한 항목 (미리 준비)

**선택 지정과제**: 9번 — 축제 수요 예측 실패 및 주관적 경험 의존형 기획으로 인한 예산 낭비·만족도 저하

**팀명**: 경사하강단

**타깃 사용자**:
- (기획자) 예산 집행 전 기획안의 객관적 수요 검증이 필요한 지자체·축제 위탁사 실무자
- (방문객) 축제 방문을 계획하는 일반 관광객

**핵심 기능**:
- 기획자 모드: 축제 조건 입력 → 예상 방문자 수·흥행 점수 예측(+근거) → 개최지·편의시설 추천 → 기획 보완 리포트 자동 생성
- 방문객 모드: 등록 축제 탐색 → 예상 혼잡도·티켓팅 경쟁도 → 주변 주차·숙소·연계 관광지 → 방문 코스 추천

**활용 데이터·API**:
- 한국관광공사 TourAPI: 행사정보(searchFestival), 지역/위치기반 관광정보, 숙박·편의시설, 이미지
- 공공데이터포털: 지역별 방문자수 빅데이터(예측 라벨)
- 외부(선택): 기상청 날씨, 공연·출연진 인기 지표
- AI: 예측모델(LightGBM 등) + 설명가능성(SHAP) + LLM 리포트

**차별점**:
- "LLM 추천 1회"에 그치는 예시들과 달리 자체 학습 예측 모델 + SHAP 설명으로 '주관적 경험 → 데이터 기반 객관 검증' 구현
- 기획자·방문객을 하나의 예측 엔진으로 연결한 양방향 순환 구조

**기대효과**: 기획자 리스크 사전 진단 / 방문객 불확실성 완화 / 지역 관광 활성화

**해시태그**: #축제/행사 기획 #사전 수요 예측 #보완 피드백 제공 #데이터 기반

## ⚠️ 반영된 결정
- "예상 리셀(암표) 값"은 개요에서 제외 → "티켓팅 경쟁도/수요 지표"로 순화. (데이터 부재 + 암표 조장 오해 방지)
- 개요는 '비전'. 실제 개발은 Tier 0(핵심 예측+기본 화면)부터 확실히 완성 — 심사는 '계획 대비 구현'을 봄.
