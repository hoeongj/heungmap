## 개발 원칙: 계층형(Tier) — 제품이 항상 작동하게
상세: `docs/03_ROADMAP_TIERS.md`
1. **Tier 0 (최우선, 제품 완성 보장)**: 규칙기반 흥행점수(ML 아님) + 데이터 조립 파이프라인 + 웹 뼈대 + TourAPI 연동. **이것만 끝나도 제출 가능한 완성작.**
2. **Tier 1**: 규칙점수를 실제 ML 모델(LightGBM)로 교체 = 팀의 첫 ML.
3. **Tier 2**: SHAP 근거 + LLM 보완 리포트 + UI 다듬기.
> 모델(Tier1)이 실패해도 Tier0가 살아있어 프로젝트가 안 망하는 구조. 이 순서를 지킬 것.
